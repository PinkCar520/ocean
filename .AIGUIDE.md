# Ocean AI 代码生产力与规约指南 (.AIGUIDE)

欢迎加入 Ocean 开发团队。本指南旨在规范 AI 辅助开发流程，确保 Monorepo 架构下的高质量代码产出。

## 1. 架构原则 (Architecture Principles)

- **全栈 DDD Monorepo**: 严格遵守 `apps/`, `packages/`, `mcp-servers/` 的三层隔离。
- **能力协议化 (MCP)**: 所有外部系统集成（禅道、GitLab 等）必须封装为 MCP Server，严禁在 Gateway 中直接编写特定系统的 HTTP 客户端。
- **业务智力化 (Skills)**: 业务逻辑应优先沉淀为 `packages/skills` 中的声明式 SOP。

## 2. 目录规范 (Directory Conventions)

- `apps/`: 包含终端用户直接交互的入口（Web, CLI, Gateway）。
- `mcp-servers/`: 纯粹的集成驱动，负责将外部接口转换为 MCP 工具。
- `packages/core`: 全局共享的类型定义、常量、工具函数。
- `packages/skills`: 存放 AI 智能体的业务指令集（Markdown 格式）。
- `docker/`: 所有的基础设施编排脚本与持久化配置。

## 3. 开发规约 (Coding Standards)

- **TypeScript**: 强制启用严格模式 (`strict: true`)。
- **Tooling**: 使用 `pnpm` 作为包管理器，严禁混合使用 `npm` 或 `yarn`。
- **Mocking**: 开发 MCP Server 时，应提供对应的 Mock 数据模式，方便脱离内网环境调试。

## 4. AI 协作声明

- 当你（AI 助手）在修改 `mcp-servers` 时，请务必同步检查 `packages/core` 中的类型定义是否需要更新。
- 所有的 Skill 编写应遵循“意图导向”，即明确：目标、步骤、异常处理分支。
